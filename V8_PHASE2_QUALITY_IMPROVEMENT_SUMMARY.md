## 🎯 V8 Phase 2의 핵심 변화

### **Before (V7)**
```
사용자 입력
    ↓
Template 채우기
    ↓
같은 구조, 같은 표현의 결과
(루테인, 베르베린, 홍삼 구분 안 됨)
```

### **After (V8 Phase 2)**
```
사용자 입력
    ↓
[Step 1] AI가 제품을 깊이 있게 분석
  ① 핵심 구매 이유?
  ② 고객 심리는?
  ③ 고객 궁금증은?
  ④ 경쟁 차별화?
  ⑤ 최적 섹션 순서?
    ↓ (분석 결과 JSON)
[Step 2] 분석 결과 기반으로 글 작성
  - 제품별 판매 전략 적용
  - 섹션별 역할 차별화
  - Template는 구조만, 내용은 새로 작성
    ↓
제품마다 완전히 다른 상세페이지
(루테인 ≠ 베르베린 ≠ 홍삼 ≠ 오메가3)
```

## 📋 수정 항목 5가지

### **1️⃣ 2단계 생성 프로세스 추가**

**파일:** `src/prompts/generationPromptV8.js` (새로 작성)

**Step 1: 제품 분석 (`buildProductAnalysisPrompt`)**
- AI가 제품을 전략적으로 분석
- 5가지 핵심 질문에 답변 (JSON 형식)
- 구매 심리, 궁금증, 차별화, 최적 순서 도출

**Step 2: 글쓰기 (`buildDetailPageGenerationPrompt`)**
- Step 1 분석 결과를 입력받음
- 분석된 전략을 실제 카피로 변환
- Template는 구조만 참고, 내용은 제품별로 다르게 작성

**효과:**
```
API 호출: 1회 → 2회 (분석 2,500 token + 글쓰기 4,000 token)
생성 시간: +20-30초 (충분한 가치 대비)
품질 향상: 매우 큼 (사람 수준으로 접근)
```

### **3️⃣ App.jsx 2단계 호출 추가**

**파일:** `src/App.jsx` (Line 1738-1770)

**변경 내용:**

```javascript
// ============================================================
// V8 Phase 2: 2단계 생성 프로세스 (분석 → 글쓰기)
// ============================================================

// STEP 1: 제품 분석 (AI가 제품을 깊이 있게 이해)
phase = "제품 분석";
setStage(1);
const analysisPrompt = buildProductAnalysisPrompt(product);
const analysisResult = await callClaude(analysisPrompt, 2500);
let analysis;
try {
  analysis = JSON.parse(analysisResult);
} catch (e) {
  console.warn("제품 분석 파싱 실패, 기본값 사용:", e.message);
  analysis = { /* 기본값 */ };
}

// STEP 2: 분석 결과를 기반으로 상세페이지 생성
phase = "상세페이지 생성";
setStage(2);
const genPrompt = buildDetailPageGenerationPrompt(product, analysis);
const genResult = await callClaude(genPrompt, 4000);
```

- ✅ UI에서 "제품 분석 중..." 상태 표시 가능

### **4️⃣ 글쓰기 지침 극대화**

**파일:** `src/prompts/generationPromptV8.js` (Line 236-264)

**강화된 지침:**

```
⭐ 가장 중요한 것:
1. 템플릿을 채우는 것이 아니라, 분석 결과를 실제 카피로 변환
2. 각 섹션이 분석된 "최적 순서"를 따르고 각자의 목표 명확히
3. 제품명이 달라지면 논리·표현·감정이 완전히 달라져야 함
4. 사람이 기획한 것처럼 자연스럽고 설득력 있게 작성

✍️ 제품 맞춤 전략 (매우 중요):
- 판매 전략: [정보 비교형 / 원료 신뢰형 / 브랜드 신뢰형 등]
- 각 섹션별 강조 방식:
  * Hero: 제품 맞춤 공감 포인트
  * Problem: 제품별로 다른 불안감
  * Solution: 제품별 차별화 방식
  * Trust: 제품별 신뢰 신호
  * Usage: 제품별 일상 활용법
```

**절대 금지:**
- ❌ 플레이스홀더 ("XXmg", "[제품명]", "~~~")
- ❌ 템플릿 설명 ("이 섹션은", "다음으로")
- ❌ 입력되지 않은 정보 생성
- ❌ 같은 문장 패턴 반복
- ❌ 섹션이 독립적으로 존재 (흐름 필수)

## 🧪 테스트 권장사항

### 테스트 시나리오

```
1. 루테인 (건강기능식품)

   확인: Hero에서 "정확한 선택" 강조하는가?
   
2. 베르베린 (일반 건강식품으로 선택했을 경우)
   예상: 원료, 배합, 신뢰 중심
   확인: Trust 섹션에서 "원료 신뢰성" 강조하는가?
   
3. 홍삼 (일반식품)
   예상: 원산지, 전통, 브랜드 신뢰
   확인: Hero에서 "신뢰" 또는 "전통" 언급하는가?
   
4. 오메가3 (건강기능식품)
   예상: 함량 명확성 + 생활 실용성
   확인: 수치와 생활 활용법이 함께 나오는가?
```

### 확인 체크리스트

```
✓ 각 제품이 다른 구조와 표현인가?
✓ 플레이스홀더가 없는가?
✓ Template 설명("이 섹션은...")이 없는가?
✓ 섹션이 자연스럽게 연결되는가?
✓ 조사 오류("가치을")가 없는가?
✓ 입력되지 않은 정보가 생성되지 않았는가?
✓ 사람이 쓴 것처럼 느껴지는가?
```
