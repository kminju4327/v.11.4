## 🔧 개선 사항 5가지

### **1️⃣ Prompt 품질 개선 (더 자연스럽고 브랜드다운 문체)**

**개선 내용:**
- 명령조 제거 → 지시 문체로 통일
- 기계적 설명 제거 → 설득력 있는 표현
- 너무 많은 마크다운 기호 제거 → 깔끔한 구조
- 문체의 일관성 강화

**예시:**
```
Before:
"이 섹션을 작성할 때 AI 설계의 강조할 특징을 포함하세요."

After:
"이전 섹션의 감정·논리를 이어받으면서 이 섹션의 목표를 명확히 달성하세요."
```

---

### **2️⃣ Product Knowledge 활용도 극대화**

**개선 내용:**
- salesStrategy → 섹션 구성 순서 결정
- buyerPsychology → 감정 표현의 기준
- topCuriosity → 필수 포함 정보
- sectionEmphasis → 각 섹션의 구체적 역할

**구현:**
```javascript
const productStrategy = `
판매 전략 유형: ${knowledge.salesStrategy}
고객 심리: ${knowledge.buyerPsychology}

각 섹션의 역할 (제품별로 다릅니다):
${Object.entries(knowledge.sectionEmphasis)
  .map(([section, emphasis]) => `
${section.toUpperCase()}:
목표: ${emphasis}
역할: ${sectionRoles[section.toLowerCase()]?.goal}
방향: ${sectionRoles[section.toLowerCase()]?.description}
`)
  .join("\n")}
`;
```

---

### **3️⃣ 섹션마다 역할 명확화**

**정의된 역할들:**

```
HERO
→ 관심을 끈다
→ 제품이 존재하는 것을 알리고, 고객의 감정을 흔든다

PROBLEM
→ 공감한다
→ 고객의 구체적인 불안감을 정확히 인식시킨다

INSIGHT
→ 깨달음을 준다
→ 왜 그 문제가 발생하는지 설명한다

INGREDIENT
→ 신뢰를 만든다
→ 원료, 배합, 제조 방식을 설명한다

QUALITY
→ 안심시킨다
→ 품질 기준, 안전성을 제시한다

SOLUTION
→ 해결책을 제시한다
→ 이 제품이 어떻게 문제를 해결하는지 설명한다

USAGE
→ 구매 후 모습을 상상하게 한다
→ 일상, 변화, 만족감을 구체적으로 그려준다

CTA
→ 자연스럽게 구매를 유도한다
→ 지금까지의 흐름을 마무리하고 구매로 이끈다
```

**프롬프트에서:**
```javascript
const sectionRoles = {
  hero: {
    goal: "관심을 끈다",
    description: "제품이 존재하는 것을 알리고, 고객의 감정을 흔든다..."
  },
  problem: {
    goal: "공감한다",
    description: "고객의 구체적인 불안감, 고민, 불편함을..."
  },
  // ... 각 섹션별로 정의
};
```

### **4️⃣ 제품별 판매 전략 확실한 차별화**

**구현:**

```javascript
섹션 구성 순서의 이유:
${knowledge.salesStrategy === "정보 비교형" ? `
1. 고객이 비교하고 싶어함 → 먼저 기능성·원료·품질을 명확히
2. 신뢰할 수 있는 정보 → 수치·기준·차이점 강조
3. 구매 결정 → 비교 우위 강조
` : knowledge.salesStrategy === "원료 신뢰형" ? `
1. 원료의 신뢰성 → 가장 먼저 원료 스토리 설명
2. 배합의 과학성 → 왜 이 원료들을 함께 넣었는지
3. 일상 루틴 → 신뢰감 바탕으로 구매 유도
` : knowledge.salesStrategy === "브랜드 신뢰형" ? `
1. 브랜드의 역사와 신뢰 → 원산지·전통·기준 강조
2. 오랜 신뢰의 근거 → 품질 관리·기준·혁신
3. 선택의 당위성 → 신뢰의 축적으로 구매 유도
` : `
1. 제품의 핵심 가치
2. 고객 혜택
3. 구매 유도
`}
```

**예시:**
```
루테인 (정보 비교형):
→ 기능성 명확 → 원료 정보 → 품질 기준 → 비교 우위 → 구매

베르베린 (원료 신뢰형):
→ 원료 스토리 → 배합 과학성 → 신뢰감 축적 → 일상 활용 → 구매

홍삼 (브랜드 신뢰형):
→ 원산지 자부심 → 전통 기준 → 세대 신뢰 → 품질 입증 → 구매
```

---

### **5️⃣ 다양성 극대화 (표현, 문장, 구조 반복 제거)**

**프롬프트 지침:**

```javascript
🎨 표현의 다양성:
- 같은 문장 구조 반복 금지
- 문단마다 다른 문장형 사용 (평서, 의문, 감탄, 선언)
- 주어를 다양하게 (제품, 고객, 우리, 상황, 감정 등)
- 설명 방식 다양화 (사실, 비유, 질문, 이야기, 수치)

✨ 브랜드 문체 목표:
- 마치 브랜드의 전담 카피라이터가 기획한 것 같음
- 제품의 개성이 느껴짐
- 고객의 심리를 정확히 파악한 느낌
- 자연스럽고 설득력 있음
```

## 📊 구조 비교

### **Before (Template 중심)**
```
프롬프트 구조:
제품정보 → Brain → Template → AI설계 → Knowledge

문제점:
- Template에 끌려 생성
- 제품별 차이 없음
- 같은 표현 반복
- AI 느낌 남음
```

### **After (제품 이해 중심)**
```
프롬프트 구조:
제품정보 
  ↓
[섹션별 역할] (HERO→관심, PROBLEM→공감, ...)
  ↓
[제품별 판매 전략] (비교형/원료신뢰형/브랜드신뢰형...)
  ↓
[Product Knowledge] (고객심리, 차별화, 궁금증)
  ↓
[AI 설계 구조]
  ↓
[표현의 다양성 가이드]
  ↓
생성

특징:
- 제품을 먼저 이해
- 섹션별 역할 명확
- 제품별 전략 명확
- 다양한 표현
- 사람 같은 느낌
```

## 💡 효과: 제품별 완전 차별화

### **루테인 (정보 비교형)**
```
판매 전략: 정보 비교형
고객 심리: 객관적 정보로 비교하려는 고객

순서:
1. HERO: "정확한 정보로 비교하고 선택"
2. PROBLEM: 선택의 어려움, 불확실성
3. INSIGHT: 왜 정보가 중요한가
4. INGREDIENT: 함량, 기능성, 명시
5. QUALITY: 식약처 인정, 차별점
6. USAGE: 효과적 섭취법
7. CTA: 신뢰하고 선택

느낌: 객관적이고 신뢰할 수 있는
```

### **베르베린 (원료 신뢰형)**
```
판매 전략: 원료 신뢰형
고객 심리: 원료와 배합의 신뢰성으로 선택

순서:
1. HERO: "신뢰할 수 있는 원료의 조합"
2. PROBLEM: 고민과 불안
3. INSIGHT: 원료가 왜 중요한가
4. INGREDIENT: 원료 스토리, 배합 이유
5. QUALITY: 배합의 과학성
6. USAGE: 일상 루틴으로 자리잡기
7. CTA: 원료를 믿고 선택

느낌: 신뢰감 있고 과학적인
```

### **홍삼 (브랜드 신뢰형)**
```
판매 전략: 브랜드 신뢰형
고객 심리: 브랜드 역사와 전통으로 신뢰

순서:
1. HERO: "세대를 거친 신뢰의 선택"
2. PROBLEM: 몸을 챙기고 싶은 마음
3. INSIGHT: 신뢰가 어디서 오는가
4. INGREDIENT: 원산지, 전통, 역사
5. QUALITY: 엄격한 기준과 관리
6. USAGE: 신뢰감 속에서 일상에 녹이기
7. CTA: 신뢰를 선택

느낌: 따뜻하고 신뢰감 있는
```

## ✅ 품질 검증

생성된 상세페이지를 평가할 때:

```
1. 제품별 차별화
   □ 루테인 vs 베르베린 완전히 다른가?
   □ Hero 헤드라인이 전략을 반영하는가?
   □ 섹션 구성이 판매 전략에 맞는가?

2. 섹션별 역할 수행
   □ HERO가 관심을 끄는가?
   □ PROBLEM이 공감을 주는가?
   □ INGREDIENT가 신뢰를 만드는가?
   □ USAGE가 미래를 상상하게 하는가?

3. 다양성
   □ 같은 문장 구조가 반복되지 않는가?
   □ 다양한 주어가 사용되는가?
   □ 다양한 설명 방식이 있는가?

4. 브랜드 느낌
   □ Template 티가 나지 않는가?
   □ AI 느낌이 아닌가?
   □ 전문가가 기획한 것처럼 느껴지는가?
   □ 제품의 개성이 느껴지는가?

5. Product Knowledge 반영
   □ 판매 전략이 명확한가?
   □ 고객 심리가 반영되어 있는가?
   □ 고객이 가장 궁금한 것이 답변되었는가?
   □ 차별화 포인트가 자연스럽게 분산되었는가?
```
