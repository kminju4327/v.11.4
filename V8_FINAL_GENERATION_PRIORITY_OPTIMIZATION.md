## 🎯 핵심 변경: 생성 우선순위

### **Before**
```
프롬프트 순서:
1. Brain 원칙 (앞)
2. Template 구조 (앞)
3. AI 설계 결과 (중간)
4. Product Knowledge (뒤)

결과: Template 구조에 끌려서 생성
```

### **After**
```
프롬프트 순서:
1. ⭐ AI 설계 결과 (Product Strategy) - 최우선
2. ⭐ Product Knowledge - 두 번째
3. Brain 원칙 - 세 번째
4. Template 레이아웃 - 네 번째

결과: 제품을 먼저 이해하고 기획해서 생성
```

## 📋 상세 변경 사항

### **1️⃣ AI 설계 결과 → 최우선 (우선순위 1)**

```javascript
[⭐ 우선순위 1: AI 설계 결과 (최우선 사용)]

📌 페이지 구조 (반드시 이 순서로):
1. 섹션1
2. 섹션2
...

💡 강조할 제품 특징:
1. 특징1
2. 특징2

🎯 설득 포인트:
1. 포인트1
2. 포인트2

📸 이미지 전략:
1. 이미지1
2. 이미지2

👥 타깃 고객:
1. 타깃1
2. 타깃2
```

**의미:**
- AI 설계에서 결정한 섹션 순서, 강조 특징, 설득 포인트를 최우선으로 사용
- 이것이 상세페이지의 기본 골격을 결정
- Template 구조보다 우선

### **2️⃣ Product Knowledge → 두 번째 (우선순위 2)**

```javascript
[⭐ 우선순위 2: Product Knowledge (두 번째 중요)]

판매 전략: ${knowledge.salesStrategy}
→ 이것이 전체 카피의 기조를 결정합니다.

고객 심리: ${knowledge.buyerPsychology}
→ 각 섹션의 감정 표현을 이것에 맞추세요.

차별화 포인트:
1. 포인트1
2. 포인트2

고객이 가장 궁금한 것:
1. 질문1
2. 질문2

톤: ${knowledge.copytone}

섹션별 강조:
HERO: ${sectionEmphasis.hero}
PROBLEM: ${sectionEmphasis.problem}
...
```

**의미:**
- 제품별 판매 전략과 고객 심리를 명시
- 각 섹션의 톤과 강조 내용을 명확히 함
- Brain과 Template 이전에 제품 고유 전략을 반영

### **3️⃣ Brain 원칙 → 세 번째 (우선순위 3)**

```javascript
[⭐ 우선순위 3: Brain 원칙 (신뢰성 기준)]

원칙:
- 과장하지 않고 확인 가능한 정보로 설득한다.

톤: ${brain.copyTone}
```

**의미:**
- 신뢰성과 법적 준수를 위한 가이드
- 1, 2순위를 신뢰성 범위 내에서 표현하는 기준
- 어떻게 표현할지를 정함

### **4️⃣ Template → 네 번째, 레이아웃만 (우선순위 4)**

```javascript
[우선순위 4: Template 레이아웃 (구조만 참고)]

섹션별 목적:
1. 섹션1: ...
2. 섹션2: ...

주의: Template의 섹션명이나 구조는 참고만 하고,
      실제 카피는 위의 1, 2, 3순위를 중심으로 새로 작성하세요.
```

**의미:**
- Template은 "몇 개의 섹션을 만들어야 하는가" 정도만 참고
- 섹션명, 구조, 내용은 AI 설계와 Product Knowledge 중심
- Template를 채우는 것이 아니라 제품을 기획하는 것

## 💡 효과: 제품별 완전 차별화

### **같은 Template, 다른 결과**

#### **루테인 (정보 비교형)**
```
우선순위 1 (AI 설계):
- 섹션: 문제 인식 → 정보 비교 → 기능성 확인 → 차별화 → 구매
- 강조: 함량, 기능성, 비교 가능성

우선순위 2 (Product Knowledge):
- 판매 전략: 정보 비교형
- 고객 심리: 객관적 정보로 비교하려는 고객
- 차별화: 함량 명확, 기능성 인정, 비교 가능

결과:
Hero: "정확한 정보로 비교하고 선택하는 눈 건강"
(Before: "눈 건강을 위한 제품")
```

#### **베르베린 (원료 신뢰형)**
```
우선순위 1 (AI 설계):
- 섹션: 고민 공감 → 원료 신뢰 → 배합 과학성 → 일상 루틴 → 구매
- 강조: 원료, 배합, 신뢰성

우선순위 2 (Product Knowledge):
- 판매 전략: 원료 신뢰형
- 고객 심리: 원료와 배합의 신뢰성으로 선택
- 차별화: 인도 자연산, 전통 의학, 과학적 배합

결과:
Hero: "신뢰할 수 있는 원료의 현명한 조합"
(Before: "건강 관리를 위한 제품")
```

## 🔧 코드 변경 사항

**파일:** `src/prompts/generationPrompt.js`

**함수:** `buildGenerationPrompt`

**변경 내용:**

1. **정보 배치 순서**
   ```
   Before: [제품정보] → [Brain] → [Template] → [AI설계] → [Knowledge]
   After:  [제품정보] → [AI설계] → [Knowledge] → [Brain] → [Template]
   ```

2. **명시적 우선순위 표시**
   ```
   [⭐ 우선순위 1: AI 설계 결과]
   [⭐ 우선순위 2: Product Knowledge]
   [⭐ 우선순위 3: Brain 원칙]
   [우선순위 4: Template 레이아웃]
   ```

3. **각 섹션 작성 지침 강화**
   ```
   ✍️ 각 섹션 작성할 때:
   - 제1순위: AI 설계의 "강조할 특징"과 "설득 포인트" 포함
   - 제2순위: Product Knowledge의 "판매 전략"과 "고객 심리" 반영
   - 제3순위: Brain의 "원칙"과 "톤" 유지
   - 제4순위: Template의 "목적"만 참고
   ```

**UI/기능:** 변경 없음
**코드 구조:** 유지 (함수명, 파라미터 동일)

## ✅ 품질 검증 항목

생성 결과를 평가할 때:

1. **제1순위 AI 설계 반영 여부**
   - [ ] 섹션 순서가 AI 설계 순서를 따르는가?
   - [ ] 강조할 특징이 모두 포함되어 있는가?
   - [ ] 설득 포인트가 자연스럽게 분산되어 있는가?

2. **제2순위 Product Knowledge 반영 여부**
   - [ ] 판매 전략이 전체 카피의 기조를 이루는가?
   - [ ] 고객 심리가 감정 표현에 반영되어 있는가?
   - [ ] 고객이 가장 궁금한 것이 답변되어 있는가?

3. **제3순위 Brain 원칙 준수 여부**
   - [ ] 신뢰성을 지켰는가?
   - [ ] 법적 제약을 준수했는가?
   - [ ] 지정된 톤이 유지되어 있는가?

4. **제4순위 Template 정상 사용 여부**
   - [ ] Template는 레이아웃만 참고했는가?
   - [ ] Template 문구를 복사하지 않았는가?
   - [ ] 섹션 수는 Template을 따랐는가?

5. **제품별 차별화 여부**
   - [ ] 루테인과 베르베린의 카피가 완전히 다른가?
   - [ ] 각 제품의 판매 전략이 명확한가?
   - [ ] Template 티가 나지 않는가?
